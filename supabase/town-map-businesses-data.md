# 동네지도 `businesses` 테이블 데이터 정의

## 문서 목적

이 문서는 `동네지도` 기능에서 사용할 `businesses` 테이블의 데이터 정의를 정리한 문서다.

이 문서는 아래 목적을 위해 사용한다.

- `businesses` 테이블 설계
- 생성형 AI를 통한 목데이터 생성
- 지도 마커, 업체 상세, 업체 게시글의 기준 엔티티 정리

이 문서는 아래 목적을 위한 문서는 아니다.

- 최종 마이그레이션 SQL
- 운영 환경용 완성 스키마
- 동네지도 전체 기능 명세

## `businesses` 테이블의 역할

`businesses` 테이블은 동네지도에 노출되는 업체의 기준 데이터다.

이 테이블은 아래 3가지 역할을 담당한다.

1. 지도 위에 업체를 마커로 표시한다.
2. 마커를 눌렀을 때 업체 상세 화면으로 이동한다.
3. 바텀시트에 보이는 업체 게시글이 어떤 업체에서 작성된 것인지 연결한다.

한 줄로 정리하면 아래와 같다.

`businesses`는 동네지도 안에서 업체를 표현하는 마스터 데이터 테이블이다.

## 권장 테이블명

- `businesses`

## 권장 필드 정의

### 1. 필수 필드

아래 필드는 최소 권장 필드다.

| 필드명                | 의미               | 역할                                          | 생성 가이드                                          |
| --------------------- | ------------------ | --------------------------------------------- | ---------------------------------------------------- |
| `id`                  | 업체 고유 식별자   | 지도, 상세, 게시글 연결의 기준 키             | `"biz_001"` 같은 문자열 또는 UUID 사용               |
| `name`                | 업체명             | 지도, 상세, 게시글에서 대표 이름으로 사용     | 실제 동네 가게처럼 자연스러운 이름 생성              |
| `category`            | 업체 카테고리      | 지도 필터, 상세 요약, 게시글 보조 정보에 사용 | 가능한 큰 분류로 통일. 예: `"음식점"`, `"카페/베이커리"`, `"생활 서비스"`, `"운동"`, `"뷰티"`, `"의료"` |
| `address`             | 기본 주소 문자열   | 상세 화면과 위치 맥락 표현에 사용             | 실제 동네 느낌이 나는 주소 문자열 생성               |
| `lat`                 | 위도               | 지도 마커 위치 표시                           | 합정역 인근 안에서만 생성                            |
| `lng`                 | 경도               | 지도 마커 위치 표시                           | 합정역 인근 안에서만 생성                            |
| `thumbnail_image_url` | 대표 썸네일 이미지 | 마커 카드, 상세 상단, 리스트에 사용           | 별도로 생성 예정이므로 비워두기                      |
| `created_at`          | 데이터 생성 시각   | 관리 및 정렬용                                | ISO datetime 사용                                    |
| `updated_at`          | 데이터 수정 시각   | 최신 상태 관리용                              | ISO datetime 사용                                    |

### 2. 업체 상세 화면용 권장 필드

아래 필드는 업체 상세 화면을 더 사실감 있게 만드는 데 중요하다.

| 필드명                | 의미             | 역할                                | 생성 가이드                        |
| --------------------- | ---------------- | ----------------------------------- | ---------------------------------- |
| `phone_number`        | 전화번호         | 상세 화면에서 연락 수단 제공        | 02-0000-0000으로 통일              |
| `road_address`        | 도로명 주소      | 상세 화면의 정식 주소로 사용        | 합정역 인근 주소로만 사용          |
| `detail_address`      | 상세 주소        | 건물, 층수, 상가 위치 정보 제공     | 예: `"1층"`, `"2층"`, `"상가 B동"` |
| `business_hours_text` | 영업시간 문자열  | 상세 화면에 바로 노출               | 예: `"매일 11:00 - 21:00"`         |
| `closed_days_text`    | 휴무 정보        | 영업 정보 보조                      | 예: `"매주 월요일 휴무"`           |
| `is_open_now`         | 현재 영업중 여부 | 지도와 상세 화면의 상태 배지에 사용 | Boolean 값 사용                    |
| `intro_text`          | 상세 소개 문구   | 업체 상세 화면 본문에 사용          | 2~4문장 정도의 자연스러운 소개     |

### 3. 현실감과 신뢰감을 높이는 권장 필드

아래 필드는 MVP에서는 선택 사항이지만, 있으면 훨씬 실제 서비스처럼 보인다.

| 필드명            | 의미                 | 역할                           | 생성 가이드                             |
| ----------------- | -------------------- | ------------------------------ | --------------------------------------- |
| `review_count`    | 리뷰 수              | 신뢰도와 인기 지표             | 업종과 가게 성격에 맞게 자연스럽게 생성 |
| `bookmark_count`  | 저장 수              | 인기 지표                      | 동네 인기 가게는 더 높게 설정 가능      |
| `view_count`      | 상세 조회 수         | 내부 지표이자 현실감 강화 요소 | 너무 극단적이지 않게 생성               |
| `distance_meters` | 기준 위치로부터 거리 | 거리 라벨 표현에 활용 가능     | 50~700m 정도 권장                       |

## 생성형 AI 데이터 생성 시 제약 조건

생성형 AI로 데이터셋을 만들 때는 아래 조건을 함께 주는 것이 좋다.

### 1. 모든 업체는 같은 생활권 안에 있어야 한다

예시:

- 합정동
- 서교동
- 망원동
- 공덕동

서로 전혀 다른 구나 생활권으로 퍼지지 않게 한다.

### 2. 좌표는 실제처럼 가까워야 한다

동네지도가 동네지도처럼 보이려면 업체 좌표가 실제로 합정역 인근 범위 안에 모여 있어야 한다.

### 3. 카테고리 분포는 자연스러워야 한다

한 카테고리만 과하게 많지 않게 구성한다.

권장 예시:

- 음식점
- 카페/베이커리
- 생활 서비스
- 운동
- 뷰티
- 의료

세부 업종은 큰 분류로 묶는 것을 권장한다.

예시:

- 돈가스, 일식, 한식, 분식, 브런치, 붕어빵은 모두 `음식점`
- 카페, 베이커리, 디저트 카페는 `카페/베이커리`
- 세탁소, 수선집, 사진관, 반려동물 미용은 `생활 서비스`

또한 카테고리는 업체 이름과 소개 문구를 봤을 때 자연스럽게 연결되어야 한다.

### 5. 영업 정보는 현실적이어야 한다

영업시간, 휴무일, 현재 영업중 여부의 조합이 자연스러워야 한다.

### 6. 인기 지표는 가게 성격에 맞게 차등을 둔다

예를 들어:

- 오래된 인기 가게는 `review_count`, `bookmark_count`가 높을 수 있다.
- 최근 생긴 가게는 상대적으로 낮을 수 있다.

## 한 줄 요약

`businesses` 테이블은 동네지도 안에서 업체를 표현하는 기준 테이블이며, 지도 마커, 업체 상세 화면, 업체 게시글 연결에 필요한 정보를 함께 담아야 한다.
