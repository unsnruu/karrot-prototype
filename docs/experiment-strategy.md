# 커뮤니티 실험 전략

## 문서 목적
이 문서는 커뮤니티 영역에서 다음 제품 실험을 준비할 때 참고할 전략 문서입니다.

이전 `동네지도` 실험은 완료된 맥락으로 보고, 현재 문서는 커뮤니티 리서치 이후 어떤 가설을 어떻게 비교 가능한 형태로 만들지 정리하는 출발점으로 사용합니다.

## 현재 상태
아직 커뮤니티 리서치가 끝나지 않았기 때문에 구체적인 실험 화면, 지표, variant는 확정하지 않습니다.

현재 단계에서는 실험안을 빨리 고정하기보다 아래를 우선합니다.

- 커뮤니티 안에서 개선할 사용자 문제를 찾는다.
- 해당 문제가 어떤 화면이나 흐름에서 발생하는지 확인한다.
- 실험 후보가 정해지면 기준안과 비교안을 작고 명확하게 나눈다.

## 핵심 질문
현재 실험 전략은 아래 질문을 중심으로 출발합니다.

`커뮤니티 안에서 어떤 사용자 문제나 진입 흐름을 개선하면 더 의미 있는 참여 행동을 만들 수 있는가?`

## 가능한 관찰 영역
리서치 결과에 따라 아래 영역을 후보로 볼 수 있습니다.

- 커뮤니티 첫 화면 탐색
- 게시글 상세 진입과 반응
- 글쓰기 FAB와 작성 유형 선택
- 동네생활 글쓰기
- 투표 글쓰기
- 모임 만들기
- 온라인 카페 만들기

## 프로토타입 구현 원칙
- 실험안이 정해지기 전까지 화면 구조를 과하게 고정하지 않는다.
- variant 간 차이는 작고 설명 가능하게 유지한다.
- 실험과 무관한 UI 변경은 최소화한다.
- 데이터 상태 차이 때문에 결과가 흔들리지 않도록 read 중심 시나리오를 유지한다.
- 실제 쓰기 저장이 필요하지 않다면 pending feature, 로컬 상태, 완료 상태 시뮬레이션으로 먼저 검증한다.

## 다음 단계
- 커뮤니티 리서치 결과 정리
- 실험 질문 1개 선정
- 기준안과 실험안 정의
- 핵심 행동 지표 정의
- 필요한 화면과 이벤트 범위 결정

## 한 문장 요약
이 문서는 커뮤니티 리서치 이후 다음 실험 가설을 작고 비교 가능한 프로토타입으로 옮기기 위한 전략 메모입니다.
