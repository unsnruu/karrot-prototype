# 홈 -> 동네지도 실험 전략

## 문서 목적
이 문서는 이 프로토타입이 최종적으로 어떤 실험 판단으로 이어져야 하는지 정리하기 위한 문서입니다.

이 문서는 아래 내용을 설명하기 위해 존재합니다.

- 왜 이 프로젝트가 A/B 테스트 준비 관점으로 해석되어야 하는지
- 무엇을 비교 대상으로 볼 것인지
- 어떤 지표로 더 좋은 UI를 판단할 것인지
- 프로토타입 구현 시 어떤 부분을 고정하고 어떤 부분만 바꿔야 하는지

이 문서는 아래 성격의 문서가 아닙니다.

- 실제 실험 플랫폼 연동 문서
- 정식 분석 이벤트 스키마 확정 문서
- 모든 화면의 UX 명세서

관련 배경은 아래 문서를 참고합니다.

- `docs/product-context.md`
- `docs/roadmap.md`
- `docs/implementation-principles.md`

## 실험의 한 줄 정의
이 프로젝트의 최종 목적은 `홈` 안에서 `동네지도`로 이어지는 여러 UI 안을 비교하고, A/B 테스트를 통해 어떤 UI가 더 높은 진입율을 만드는지 판단하는 것입니다.

## 핵심 질문
이 실험이 답하려는 질문은 아래와 같습니다.

`홈 흐름 안에서 어떤 UI가 사용자를 가장 자연스럽게 동네지도로 유도하는가?`

## 비교 대상
비교 대상은 기본적으로 `홈 -> 동네지도` 진입을 유도하는 UI surface 입니다.

예시는 아래와 같습니다.

- 홈 상단 퀵 액션
- 홈 피드 중간 삽입 모듈
- 상품 카드 사이 추천 블록
- 상품 상세 안의 위치 기반 CTA

중요한 점은 “동네지도를 보여주는가”가 아니라 “어떤 UI 노출 방식이 더 높은 진입 행동을 만드는가”입니다.

## 기본 실험 단위
초기 실험 단위는 아래처럼 단순하게 유지하는 것이 좋습니다.

- Variant A: 현재 또는 기준안
- Variant B: 새 진입 UI 안

필요하면 이후 A/B/n 으로 확장할 수 있지만, 첫 단계에서는 비교 해석이 쉬운 2안 구조가 적절합니다.

## 핵심 판단 지표
가장 중요한 1차 지표는 아래입니다.

- `동네지도 진입율`

현재 문서 맥락에서 진입율은 아래처럼 해석합니다.

- 특정 홈 진입 surface를 본 사용자 중
- 실제로 `동네지도`로 이동한 비율

보조 지표는 필요 시 아래를 참고할 수 있습니다.

- CTA 클릭률
- 홈 내 해당 모듈 노출 대비 클릭률
- 동네지도 진입 후 체류 또는 후속 탐색 여부

다만 문서 기준의 north star 는 어디까지나 `동네지도 진입율`입니다.

## 프로토타입 구현 원칙
비교 가능한 실험으로 이어지려면 프로토타입은 아래 원칙을 따라야 합니다.

- variation 간 핵심 차이만 바꾼다.
- 홈의 나머지 구조와 맥락은 가능하면 동일하게 유지한다.
- 실험과 무관한 UI 차이는 최대한 줄인다.
- 데이터 상태 차이 때문에 결과가 흔들리지 않게 read 중심 시나리오를 유지한다.

즉, 이 프로젝트에서 프로토타입은 “보기 좋은 시안”보다 “비교 가능한 실험 환경”에 더 가깝게 해석해야 합니다.

## 구현 시 체크 질문
새 UI 안을 만들 때는 아래를 먼저 확인합니다.

- 이 변화가 실제로 `홈 -> 동네지도` 진입에 영향을 줄 핵심 변화인가?
- 비교안과 달라지는 요소가 너무 많지는 않은가?
- 실험과 무관한 레이아웃 변화가 섞여 있지는 않은가?
- 나중에 A/B 테스트 조건으로 분기하기 쉬운 구조인가?

## 현재 권장 첫 실험
현재 기준에서 가장 유력한 첫 실험은 아래 방향입니다.

- `홈 피드 안의 맥락형 진입 포인트`를 기준안과 비교한다.

이 방향이 좋은 이유는 아래와 같습니다.

- 기존 홈 탐색 흐름을 크게 깨지 않는다.
- `동네지도`를 별도 탭이 아니라 탐색의 자연스러운 연장선으로 제안할 수 있다.
- A/B 비교 시 차이를 설명하기가 비교적 쉽다.

## 성공 판단
초기 문서 기준에서 성공은 아래처럼 해석합니다.

- 기준안 대비 새로운 UI 안이 더 높은 `동네지도` 진입율을 만든다.
- 그 차이가 실험 관점에서 설명 가능하다.
- 결과를 바탕으로 다음 variation 을 설계할 수 있다.

## 한 문장 요약
이 프로젝트는 `홈 -> 동네지도` 진입 UI를 비교 가능한 형태로 구현하고, 최종적으로 A/B 테스트를 통해 더 높은 진입율을 만드는 안을 찾기 위한 프로토타입입니다.
