# Commit Conventions

## 기본 형식

`Type: 한줄 요약`

필요하면 아래에 한국어 상세 설명을 추가합니다.

## Type 규칙

- `Feat` 기능 추가 또는 사용자 행동 변화가 있는 작업
- `Fix` 버그 수정
- `Refactor` 동작 변화 없이 구조만 정리하는 작업
- `Style` UI 다듬기, 스타일 수정, 포맷 정리
- `Docs` 문서 작업
- `Test` 테스트 추가 또는 수정
- `Chore` 설정, 유지보수, 툴링 작업

## 작성 규칙

- `Type` 은 영어로 유지합니다.
- 제목과 상세 설명은 한국어로 작성합니다.
- 한 커밋에는 하나의 논리적 변경만 담습니다.
- 제목은 짧고 바로 이해되게 작성합니다.
- 상세 설명은 필요할 때만 작성합니다.

## 예시

```text
Feat: 홈에서 동네지도 진입 카드 추가

- 피드 중간에 지도 진입 CTA 카드 추가
- CTA 클릭 시 동네지도 화면으로 이동
- 모바일 기준 간격 조정
```

## 피해야 할 표현

- `Feat: 수정`
- `Fix: 이것저것 반영`
- `Chore: 변경사항 적용`
