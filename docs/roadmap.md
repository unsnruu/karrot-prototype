# 로드맵과 향후 작업

## 문서 목적
이 문서는 현재 단계에서 남겨둔 작업, 제품 부채, 디자인 시스템 정렬 메모를 가볍게 추적하기 위한 문서입니다.

이전 실험 중심 로드맵은 종료된 맥락으로 보고, 앞으로는 기본 앱 경험을 유지하기 위한 작업을 짧게 남기는 용도로 사용합니다.

## 현재 초점
현재 프로젝트의 초점은 실험용 분기 없이 기본 당근 앱 경험을 안정적으로 유지하는 것입니다.

## 향후 작업
### 커뮤니티 글쓰기 플로우
- 커뮤니티 탭에는 글쓰기 FAB와 글쓰기 유형 선택 바텀시트가 추가되어 있다.
- 현재 바텀시트의 각 항목은 실제 작성 화면으로 이어지지 않고 pending feature로 연결되어 있다.
- 추후 `이웃에게 투표받기`, `동네생활 글쓰기`, `모임 만들기`, `온라인 카페 만들기` 작성 플로우를 설계하고 구현해야 한다.
- 최소 구현 시에는 작성 화면, 입력값 검증, 등록 완료 상태, 커뮤니티 목록 복귀 흐름을 함께 고려한다.

### 기본 앱 경험 유지
- 커뮤니티 글 목록은 80개씩 자동으로 추가 노출한다.
- 새 화면을 만들 때는 기존 하단 탭, 상세 화면, pending feature 흐름과 자연스럽게 연결한다.
- 실험용 강제 진입 route나 variant별 화면은 추가하지 않는다.

## SEED Design 정렬 메모
프로젝트는 당근의 SEED Design과 점진적으로 가까워지는 방향을 유지합니다.

현재는 이미 아래 공용 프리미티브가 정리되어 있습니다.

- `ActionButton`
- `FieldButton`
- `UserAvatar`
- `SelectionChip`
- `Tabs`
- `ListSection`
- `PageHeader`
- `AppToolbar`
- `IconButton`

앞으로의 원칙은 아래와 같습니다.

- SEED가 제공하는 foundation, component, snippet을 우선 기준으로 삼는다.
- 로컬 프리미티브는 독자 디자인 시스템이 아니라 SEED로 이행하기 위한 임시 정리층으로 본다.
- 새 공용 컴포넌트는 SEED 개념과 충돌하지 않게 만든다.
- 토큰 체계를 새로 키우기보다 SEED 도입 경로를 먼저 확인한다.
