# 당근 프로토타입 컨텍스트

## 문서 목적
이 문서는 프로젝트의 현재 제품 맥락과 실험 방향을 공유하기 위한 문서입니다.

구체적인 구현 순서와 남은 작업은 `docs/roadmap.md`를 기준으로 봅니다.

## 현재 상태
이전 프로젝트 목표였던 `동네지도` 진입 실험은 완료되었고, 결과까지 확인된 상태입니다.

이제 프로젝트의 관심사는 `커뮤니티` 영역으로 이동했습니다. 현재는 사용자가 커뮤니티에 최초 진입할 때 관심 주제를 선택하게 하고, 해당 주제 중심으로 추천 피드를 구성했을 때 관련성 판단과 참여 행동이 개선되는지 검증하는 실험을 준비합니다.

## 현재 프로젝트 목표
이번 단계의 목표는 커뮤니티 최초 진입 시 관심 주제를 수집하고, 선택한 주제 중심의 추천 피드가 글 상세 진입 이후 참여 행동에 어떤 영향을 주는지 확인하는 것입니다.

현재는 아래 질문을 중심으로 봅니다.

`커뮤니티 최초 진입 시 관심 주제를 물어보고 그 주제 중심으로 피드를 보여주면, 사용자가 나와 관련 있는 글을 더 잘 찾고 참여할 수 있는가?`

## 제품 방향
- 당근과 유사한 모바일 앱 경험은 유지합니다.
- 모든 기능을 완성하기보다, 커뮤니티 실험에 필요한 화면과 흐름을 우선합니다.
- 커뮤니티 관심 주제 추천 실험은 `control`, `interest_based` variant를 비교합니다.
- `/`로 들어온 사용자는 중고거래 홈이 아니라 커뮤니티 탭으로 먼저 진입합니다.
- 커뮤니티 글 목록은 자동 infinite load로 80개씩 추가 노출합니다.
- 커뮤니티 글쓰기, 모임, 카페, 게시글 상세 같은 흐름은 리서치 결과에 따라 깊이를 조절합니다.

## 테스트 환경
검증과 사용성 테스트는 모바일 경험을 우선합니다.

다만 모바일 폭에만 고정된 앱으로 만들지는 않습니다. 큰 화면에서도 사용할 수 있고, 레이아웃이 깨지지 않는 반응형 구조를 유지합니다.

## 범위 메모
이 프로젝트는 완성형 서비스가 아니라 실험 준비용 프로토타입입니다.

따라서 실제 운영 수준의 CRUD, 인증, 권한, 실시간 동기화는 현재 단계의 기본 요구사항이 아닙니다. 필요한 경우에는 로컬 상태, pending feature, read 중심 데이터로도 충분히 시연 가능한 흐름을 먼저 만듭니다.

## 한 문장 요약
이 프로젝트는 `동네지도` 실험을 마치고, 이제 `커뮤니티` 관심 주제 추천 실험을 검증하기 위한 모바일 우선 프로토타입입니다.
